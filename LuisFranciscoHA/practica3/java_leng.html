¿Cuáles son las características de Java?

  1. Orientado a objetos: Utiliza un paradigma que ayuda a manejar la complejidad del software.
  2. Portable: Permite ejecutar el mismo programa en diferentes sistemas operativos.
  3. Seguro: Cuenta con un modelo de seguridad que protege los sistemas contra amenazas y vulnerabilidades.
  4. Robusto: Mantiene la integridad de los datos y previene errores.
  5. Multitarea: Permite realizar varias tareas al mismo tiempo.
  6. Distribuido: Permite trabajar en red y ejecutar código en sistemas remotos.
  7. Dinámico: Se adapta a las necesidades del usuario.
  8. Escalable: Se puede adaptar a las necesidades de crecimiento de la aplicación.
  9. Compatible con bibliotecas: Cuenta con bibliotecas estándar para funciones comunes.

Tipos de datos

Tipos de datos primitivos:

  1. byte: Ocupa 8 bits y su rango de valores es de -128 a 127.
  2. short: Ocupa 16 bits.
  3. int: Ocupa 32 bits.
  4. long: Ocupa 64 bits.
  5. float: Ocupa 32 bits.
  6. double: Ocupa 64 bits.
  7. boolean: Sus valores son true o false.
  8. char: Ocupa 16 bits.

Tipos de datos de referencia:

Los datos de referencia son tipos de datos más complejos que hacen referencia a objetos almacenados en memoria. No almacenan directamente el valor, sino una referencia a la ubicación en memoria donde se encuentra el objeto.

Otros tipos de datos de Java:

  • java.lang.String
  • java.util.Date
  • java.util.Calendar
  • java.sql.Date
  • java.sql.Time
  • java.sql.Timestamp
  • java.math.BigInteger
  • java.math.BigDecimal

Ventajas y desventajas

Ventajas:

  1. Portabilidad: "Write Once, Run Anywhere" gracias a la JVM.
  2. Orientación a Objetos: Fomenta la modularidad y reutilización de código.
  3. Gran Comunidad y Ecosistema: Amplio soporte de bibliotecas, herramientas y frameworks.
  4. Gestión Automática de Memoria: Recolector de basura para la gestión de memoria.
  5. Seguridad: Características integradas para proteger contra código malicioso.
  6. Soporte Multihilo: Habilidad para manejar procesos concurrentes eficientemente.
  7. Compatibilidad hacia Atrás: Código antiguo sigue funcionando en versiones nuevas de Java.

Desventajas:

  1. Rendimiento Inferior: Comparado con lenguajes compilados como C++.
  2. Uso de Memoria Alto: Requiere más recursos de memoria debido a la JVM.
  3. Compilación Lenta: Puede ser más lenta debido a la conversión a bytecode.
  4. Curva de Aprendizaje: Conceptos complejos como clases, objetos y referencias.
  5. Exceso de Verbosidad: Se necesitan más líneas de código para realizar tareas simples.
  6. Dependencia de la JVM: Necesitas tener la JVM instalada para ejecutar programas.
  7. No adecuado para aplicaciones de bajo nivel: No ideal para tareas que requieren control detallado de hardware.
