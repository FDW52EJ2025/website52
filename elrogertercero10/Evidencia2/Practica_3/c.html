A grandes rasgos, ¿qué es C?

El lenguaje de programación C es un lenguaje de programación de bajo nivel y de propósito general, desarrollado por Dennis Ritchie en 1972 en los Laboratorios Bell. C es conocido por su eficiencia, control detallado de la memoria y su portabilidad entre plataformas. Aunque es un lenguaje de bajo nivel, C se utiliza ampliamente para desarrollar sistemas operativos, software embebido y aplicaciones de alto rendimiento.

¿Qué hace diferente a C del resto de lenguajes de programación?

Lenguaje estructurado:
C sigue el paradigma de programación estructurada, lo que significa que el flujo del programa se organiza mediante funciones.
Biblioteca estándar:
C ofrece una biblioteca estándar rica en funciones, que incluye operaciones de entrada/salida, manipulación de cadenas, matemáticas, etc.
Uso de punteros:
C permite el uso de punteros, lo que otorga un control directo sobre la memoria.
Lenguaje de bajo nivel:
Ofrece un control detallado sobre los recursos del sistema (memoria, hardware), lo que permite optimizar el rendimiento.

Tipos de datos

  • Número
      • int: Enteros.
      float: Números con punto flotante de precisión simple.
      double: Números con punto flotante de doble precisión.
      long: Enteros más grandes.
  • char (Caracteres): Un sólo caracter.
  • Valores booleanos en C En C, no existe un tipo bool nativo como en otros lenguajes, pero los valores 0 y 1 se usan comúnmente para representar valores falsos y verdaderos, respectivamente.
  • pointer (Punteros): Variables que almacenan la dirección de memoria de otras variables.

Ventajas y desventajas de C

Ventajas

Eficiencia:
C es un lenguaje muy eficiente en términos de tiempo de ejecución y uso de recursos, lo que lo hace ideal para sistemas operativos, software embebido y aplicaciones de alto rendimiento.
Fundamento de otros lenguajes:
Muchos lenguajes modernos, como C++, Java y Python, están basados en o influenciados por C, por lo que aprender C puede ser un excelente punto de partida.

Desventajas

No tiene soporte nativo para programación orientada a objetos:
A diferencia de C++, Java y otros lenguajes modernos, C no soporta directamente conceptos como herencia y polimorfismo.
Falta de manejo de excepciones:
C no tiene mecanismos nativos para manejar errores de manera estructurada, lo que hace que el manejo de errores sea más propenso a errores y más difícil de gestionar en programas grandes.
