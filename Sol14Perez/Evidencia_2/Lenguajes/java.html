¿Qué es Java?

Java es un lenguaje de programación de alto nivel , orientado aes un lenguaje de programación de alto nivel, orientado a objetos y basado en clases, desarrollado por James Gosling y su equipo eny su equipo en Sun Microsystems ( luego adquirida por Oracle ) en 1991 . Java(luego adquirido por Oracle) en 1991. Java fue diseñado con el objetivo de ser un lenguaje que pudiera ejecutarse en diferentes plataformas sin necesidad de recompilar el código, lo que llevó a la creación del famoso lema "Write Once, Run Anywhere" (WORA)sistemas empresariales .. Gracias a su robustez, seguridad, portabilidad y facilidad de uso, Java ha sido ampliamente adoptado en desarrollo web, aplicaciones móviles (especialmente para Android) y sistemas empresariales.

Características de Java

  • Independencia de la plataforma (WORA): La principal característica que define a Java es la Máquina Virtual Java (JVM) . El código Java se compila en bytecode, que puede ser ejecutado en cualquier plataforma que tenga una JVM instalada, sin necesidad de recompilar el código.
  • Sintaxis similar a C/C++: Java tiene una sintaxis que es similar a C y C++, lo que lo hace familiar para muchos programadores. Sin embargo, Java elimina algunas de las complejidades y peligros de estos lenguajes (por ejemplo, punteros).
  • Garbage Collection (Recolección de basura): Java maneja la gestión de memoria automáticamente a través de un sistema de recolección de basura, lo que reduce la posibilidad de errores como fugas de memoria y facilita la gestión eficiente de la memoria en las aplicaciones.
  • Tipado estático y fuerte: Java es un lenguaje de tipado estático, lo que significa que las variables deben ser declaradas con un tipo específico. Esto ayuda a prevenir errores en el tiempo de ejecución y facilitar la detección de errores durante la compilación.
  • Biblioteca estándar rica: Java proporciona una extensa biblioteca estándar, conocida como Java API , que incluye clases para trabajar con colecciones de datos, interfaces gráficas, redes, bases de datos, y más.
  • Compilado y ejecutado en la JVM: El código fuente de Java se compila en un bytecode intermedio, que es interpretado y ejecutado por la JVM en lugar de ser ejecutado directamente en el sistema operativo, lo que permite su ejecución en Múltiples plataformas.

Tipos de Datos

Datos primitivos

byte Tipo de dato entero de 8 bits con signo (rango de -128 a 127).

short Tipo de dato entero de 16 bits con signo.

int Tipo de dato entero de 32 bits con signo.

long Tipo de dato entero de 64 bits con signo.

float Tipo de dato de punto flotante de 32 bits, utilizado para representar números decimales con precisión simple.

double Tipo de dato de punto flotante de 64 bits, utilizado para representar números decimales con precisión doble.

charTipo de dato que almacena un solo carácter Unicode de 16 bits.

boolean Tipo de dato para valores lógicos, true o false.

Datos de referencia

String Tipo de dato para cadenas de texto.

array Estructura de datos que almacena elementos del mismo tipo. En Java, los arreglos son objetos.

Object Tipo base de todos los objetos en Java. Todas las clases en Java heredan de Object.

Datos avanzados

enum Tipo que define un conjunto de constantes con nombre, útil para representar un conjunto de valores predefinidos.

interface Define un contrato que las clases pueden implementar.

class Define tipos de datos personalizados, que pueden contener tanto atributos (variables) como métodos.

Ventajas y Desventajas de C

Ventajas

Portabilidad (WORA - Write Once, Run Anywhere) Java se ejecuta en cualquier dispositivo o plataforma que tenga la Java Virtual Machine (JVM) instalada. El código compilado en bytecode puede ejecutarse en cualquier sistema operativo sin necesidad de recompilación.

Gestión automática de memoria Java cuenta con un sistema de recolección de basura, lo que significa que la memoria se maneja automáticamente, reduciendo la posibilidad de errores como fugas de memoria, que son comunes en otros lenguajes como C o C++.

Seguridad Java tiene un modelo de seguridad robusto, gracias a la JVM, que impide que los programas maliciosos puedan acceder a los recursos del sistema de forma no autorizada. Además, tiene características como la verificación de bytecode y la ejecución en un entorno aislado (sandboxing).

Orientación a objetos Java es completamente orientado a objetos, lo que facilita la organización del código, la reutilización y la modularidad. Los programas Java se componen de clases y objetos.

Multihilo Java tiene soporte nativo para la programación multihilo, lo que permite ejecutar varias tareas simultáneamente, aprovechando los procesadores multi-core de manera eficiente.

Desventajas

Rendimiento Aunque Java es generalmente rápido, el hecho de que sea un lenguaje interpretado (a través de la JVM) puede hacer que las aplicaciones Java sean más lentas que las compiladas directamente a código de máquina, como C o C++. La JVM introduce una capa adicional que puede afectar el rendimiento.

Consumo de memoria Las aplicaciones Java pueden consumir más memoria que las aplicaciones escritas en otros lenguajes como C debido a la necesidad de ejecutar la JVM y la recolección de basura.

Curva de aprendizaje Aunque Java tiene una sintaxis relativamente sencilla, su modelo de objetos y algunas de sus características más avanzadas (como la gestión de excepciones y multihilos) pueden ser difíciles de aprender para los principiantes.

Código más verboso Java tiende a ser más verboso que otros lenguajes modernos. Por ejemplo, muchas veces se necesita escribir más código que en lenguajes como Python o C# para lograr lo mismo, lo que puede afectar la productividad en ciertos contextos.
