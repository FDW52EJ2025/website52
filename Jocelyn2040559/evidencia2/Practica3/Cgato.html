C#

Informacion general

C# (C Sharp) es un lenguaje de programación moderno, orientado a objetos y de propósito general, desarrollado por Microsoft a principios de los años 2000 como parte de su plataforma .NET. Su sintaxis se inspira en C, C++ y Java, combinando potencia con facilidad de uso. Está diseñado para ser seguro, robusto y productivo, siendo muy usado en aplicaciones de escritorio, desarrollo web, servicios en la nube, e incluso videojuegos (especialmente con el motor Unity). Es un lenguaje fuertemente tipado y se ejecuta sobre la Common Language Runtime (CLR), similar a la JVM de Java.

C# es una excelente opción si trabajás en entornos empresariales, desarrollo de interfaces gráficas, servicios web, o juegos. Su integración con herramientas como Visual Studio y Azure lo hace muy potente en el mundo corporativo.

Caracteristicas Principales

  1. - Lenguaje orientado a objetos, con soporte también para programación funcional y eventos.
  2. - Compilado e interpretado a través de la CLR.
  3. - Ofrece gestión automática de memoria con recolector de basura.
  4. - Tiene una sintaxis limpia y moderna, fácil de leer.
  5. - Soporte nativo para interfaces gráficas (WPF, WinForms) y desarrollo web (ASP.NET).
  6. - Integración fluida con el ecosistema de Microsoft (Windows, Azure, Visual Studio, etc.).

Tipos de datos

  1. Primitivos: int, float, double, char, bool, decimal, byte, long, short
  2. Compuestos y estructurados: string, object, array, enum, struct, class, interface
  3. Colecciones: List, Dictionary, Queue, HashSet

Ventajas y Desventajas

Ventajas Desventajas
Lenguaje moderno, potente y fácil de mantener. Inicialmente limitado a sistemas Windows, aunque eso ha cambiado con .NET Core y .NET 5+.
Ideal para desarrollo de videojuegos con Unity. Puede requerir más recursos en comparación con lenguajes más livianos como C o Go.
Tiene características avanzadas como LINQ, async/await, manejo de eventos y delegados. Algunas bibliotecas y tecnologías aún dependen del ecosistema Windows.
Gestión automática de memoria, minimizando errores comunes. Compilación más lenta que lenguajes interpretados en ciclos de desarrollo rápido.
