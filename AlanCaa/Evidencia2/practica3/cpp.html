Historia

El lenguaje C++ fue creado por Bjarne Stroustrup en 1979 en los Laboratorios Bell, como una extensión del lenguaje de programación C. La intención de Stroustrup era añadir características de orientación a objetos al lenguaje C, mientras mantenía su eficiencia y rendimiento. Inicialmente, el lenguaje fue llamado "C con clases", pero más tarde se le renombró como C++ en 1983, haciendo referencia al incremento de la versión en el lenguaje C. C++ se ha convertido en uno de los lenguajes más populares en el desarrollo de software, especialmente en sistemas de alto rendimiento, videojuegos, y aplicaciones en tiempo real.

Características Principales

Lenguaje compilado
C++ es un lenguaje compilado, lo que significa que el código fuente debe ser transformado en código máquina antes de su ejecución, lo que generalmente resulta en un rendimiento mucho mayor en comparación con los lenguajes interpretados.
Soporte para programación orientada a objetos
C++ permite la programación orientada a objetos (OOP), facilitando el diseño modular y la reutilización del código mediante el uso de clases y objetos. Además, soporta la herencia, polimorfismo, encapsulamiento y abstracción.
Control de bajo nivel
Gracias a su cercanía con el lenguaje C, C++ permite un control detallado sobre la memoria y los recursos del sistema, lo que lo convierte en una excelente opción para aplicaciones que requieren un alto rendimiento y manipulación directa del hardware.
Lenguaje de alto rendimiento
C++ es conocido por su velocidad y eficiencia, lo que lo hace ideal para sistemas que requieren procesamiento intensivo, como videojuegos, simulaciones y aplicaciones de software embebido.
Soporte para plantillas (templates)
C++ introduce el concepto de plantillas, que permite escribir funciones y clases genéricas, aumentando la flexibilidad y reutilización del código sin sacrificar el rendimiento.

Tipos de datos

  • int: Números enteros.
  • float: Números con decimales de precisión simple.
  • double: Números con decimales de precisión doble.
  • char: Carácter único.
  • bool: Valores booleanos, puede ser true o false.
  • string: Secuencias de caracteres (aunque C++ no tiene tipo string nativo, se puede usar la clase `std::string`).
  • array: Colección de elementos del mismo tipo con un tamaño fijo.
  • struct: Estructura de datos que agrupa diferentes tipos de datos bajo un mismo nombre.

Ventajas y Desventajas al usar C++

Ventajas de C++:

  • Rendimiento excepcional: C++ es conocido por su alta eficiencia y velocidad.
  • Control total sobre los recursos del sistema: Permite un manejo directo de la memoria y hardware, lo que es esencial en aplicaciones de sistemas y programación embebida.
  • Versatilidad: Se utiliza en una amplia gama de aplicaciones, desde videojuegos hasta sistemas operativos.
  • Bibliotecas estándar potentes: La STL (Standard Template Library) proporciona herramientas como vectores, listas, mapas, etc.
  • Lenguaje multiparadigma: C++ soporta tanto la programación orientada a objetos como la programación estructurada, y también puede usarse para programación genérica con plantillas.

Desventajas de C++:

  • Complejidad: La sintaxis y las características avanzadas del lenguaje (como las plantillas y la gestión manual de memoria) pueden resultar difíciles de aprender y manejar para los principiantes.
  • Gestión manual de memoria: A diferencia de otros lenguajes modernos, C++ no tiene recolección de basura, lo que obliga a los programadores a gestionar la memoria manualmente, lo que puede dar lugar a errores como fugas de memoria.
  • Compatibilidad limitada con la plataforma: Aunque es multipropósito, algunas características de C++ pueden no ser compatibles con ciertos sistemas o plataformas, lo que puede complicar la portabilidad del código.
  • Tiempo de compilación largo: Los programas en C++ tienden a tener tiempos de compilación largos, especialmente cuando se utilizan muchas plantillas o bibliotecas complejas.
  • Mayor posibilidad de errores: El alto control sobre los detalles de bajo nivel en C++ puede dar lugar a errores difíciles de rastrear y corregir.
